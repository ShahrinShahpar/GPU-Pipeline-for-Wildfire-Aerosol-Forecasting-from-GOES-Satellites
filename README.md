# GPU Pipeline for Wildfire Aerosol Forecasting from GOES Satellites

A fully GPU-accelerated pipeline for processing GOES-16/17 satellite AOD (Aerosol Optical Depth) data and running a physics-informed Bayesian state-space model for wildfire smoke forecasting. All computation runs on GPU (CUDA); only data download and preprocessing use lightweight Python.

---

## Science Background

- **Satellites:** GOES-16 (75 deg W) and GOES-17 (137 deg W)
- **Product:** ABI-L2-AODC (Aerosol Optical Depth, CONUS)
- **Date:** October 1, 2020 (day 275), hours 18-22 UTC (60 snapshots each satellite)
- **Study domain:** lon [-124, -121.6], lat [35.0, 37.4] -- northern California
- **Grid:** 60x60 cells at 0.04 deg resolution = 3,600 cells


## Step-by-Step Details

### Step 1 -- `01_download_data.py`

Downloads all raw NetCDF files from NOAA's public AWS S3 archive. No credentials needed.

- **Input:** Internet connection
- **Output:** `data/AOD/GOES-16/275/` and `data/AOD/GOES-17/275/` -- 60 `.nc` files each
- **Runtime:** ~3-5 minutes

```bash
python3 01_download_data.py
```

**Verify:**
```bash
find data/AOD/GOES-16/275 -name "*.nc" | wc -l   # expect 60
find data/AOD/GOES-17/275 -name "*.nc" | wc -l   # expect 59 (one file missing in NOAA archive)
```

---

### Step 2 -- `02_coordinate.py`

Converts the satellite's fixed scan-angle grid (1500x2500 pixels) into geographic lat/lon coordinates using the geostationary projection. Run once -- coordinates are static.

- **Input:** First NC file for each satellite (reads projection parameters)
- **Output:** `data/AOD/g16_lats.txt`, `g16_lons.txt`, `g17_lats.txt`, `g17_lons.txt` (~28 MB each)
- **Runtime:** Under 30 seconds
- **Requires:** pyproj + netCDF4

```bash
python3 02_coordinate.py
```

---

### Step 3 -- `03_preprocess.py`

For each of the 120 NC files: reads AOD, applies masking and scaling, pairs each pixel with its lat/lon, filters to the study domain, clips negative AOD to 0, writes a compact binary file for the GPU.

- **Input:** NC files + lat/lon lookup tables
- **Output:** `bin_images/G16/img_001.bin ... img_060.bin`, `bin_images/G17/...`, `manifest_G16.txt`, `manifest_G17.txt`
- **Binary format:** `int32 n` | `float64[n] lats` | `float64[n] lons` | `float64[n] aods`
- **Runtime:** ~1-2 minutes

```bash
python3 03_preprocess.py
```

**Verify:**
```bash
wc -l manifest_G16.txt   # expect 60
wc -l manifest_G17.txt   # expect 59
```

---

### Steps 4-6 -- `aod_pipeline` (GPU)

All GPU layers are chained in `main_pipeline.cu`. Data stays on device between stages -- no host-device round trips between regrid, KNN, optical flow, G matrix, and Gibbs.

Must be compiled and run on a GPU node (use `srun -p gpu06` or equivalent).

```bash
make aod_pipeline        # compile (Makefile auto-detects GPU arch via nvidia-smi)
./aod_pipeline           # runs full pipeline for G16 and G17
```

The pipeline runs these passes in order:
1. **KNN** -- regrid scattered point clouds onto 60x60 grid, KNN-impute NaN cells (G16 + G17)
2. **Optical flow** -- 20 image pairs, average pairs 1-19, cuFFT image smooth (G16 only)
3. **Diffusivity K** -- Smagorinsky model: v_a -> K -> K_smooth -> dK/dx, dK/dy
4. **G matrix** -- Fourier basis Omega(N=20) -> G_ad Phi integrals -> expm(G_gen)
5. **Gibbs FFBS2** -- F matrix, UDS downsampling, obs assembly, Kalman filter + backward sampling

Timing is saved to `output/timing_gpu.csv`.
